About This App
Buffett Value Score™ combines 8 value principles:

1. P/E below industry average
2. P/B below 1.5x book value
3. ROE consistently above 15%
4. Low debt-to-equity ratio
5. Consistent earnings growth (5–10yr)
6. Wide economic moat (brand/IP)
7. Free cash flow positive
8. Margin of safety ≥ 25%
How picks are generated:

Claude AI is prompted to apply Buffett's methodology from The Intelligent Investor and Berkshire letters, reasoning about current market conditions and valuation across screened tickers. Results are AI-generated analysis, not financial advice.

GitHub Deployment Guide

This app is a single index.html file — no build step, no dependencies. Here's how to get it live on GitHub Pages in under 5 minutes.

  1. Create a GitHub repository
    Go to github.com/new and create a new repository. Name it something like oracle-buffet. Set it to Public (required for free GitHub Pages). Check "Add a README".
  2. Upload index.html
    Click "Add file → Upload files" on your new repo. Drag and drop the index.html file you downloaded. Commit directly to main.
  3. Enable GitHub Pages
    Go to Settings → Pages. Under "Source", select main branch, root folder / (root). Click Save. GitHub will deploy and give you a live URL like:
    https://yourusername.github.io/oracle-buffet
  4. Handle your API key securely
    Since this is a static site, your API key is entered at runtime in the app's Settings tab and stored in your browser's localStorage — it is never committed to GitHub. Each user who visits will enter their own key. This is the standard approach for single-user static tools.
    # Never do this:
git add secrets.js   # ❌ exposes your key publicly

# Safe pattern (already implemented):
# Key lives only in the user's browser localStorage
  5. Optional: Add a custom domain
    In Settings → Pages, add your domain under "Custom domain". Create a CNAME DNS record pointing to yourusername.github.io. Free HTTPS is automatic.
  6. Update the app
    To update later, simply replace index.html in your repo. GitHub Pages re-deploys automatically within ~1 minute. Or use Git locally:
    git clone https://github.com/yourname/oracle-buffet
# edit index.html
git add index.html
git commit -m "update screener filters"
git push
💡 Pro tip: Add a README.md that describes the app and mentions it requires a personal Anthropic API key to function. This way if others find your repo, they understand the setup.
